## 1. Design Principles

These are not aspirations — they are constraints.

| Principle | What it means in practice |
|---|---|
| **Minimal, not empty** | Every element earns its space. No decorative elements without purpose. |
| **Information hierarchy first** | The most critical information is always the most visually dominant. |
| **Professional, not decorative** | Subtle depth and texture — not gradients for the sake of gradients. |
| **Fast to scan** | A trained staff member should read a dashboard in under 5 seconds. |
| **Consistent over creative** | Use existing components. Never invent a new pattern when one exists. |
| **Keyboard-first operations** | Every workflow must be completable without a mouse. |
| **Accessible contrast** | WCAG AA minimum everywhere; AAA on critical data. |
| **Responsive across devices** | Desktop-primary for operations; tablet support for housekeeping and POS. |

## 5. Elevation & Shadow System

Used in dark mode — shadows are expressed as border highlights and layered backgrounds.

| Level | Visual Treatment | Usage |
|---|---|---|
| **0** (base) | `bg-brand-deep` — no shadow | App shell background |
| **1** (surface) | `bg-brand-surface` | Cards, sidebar, panels |
| **2** (elevated) | `bg-brand-surface-2` + `border border-subtle` | Modals, dropdowns, popovers |
| **3** (floating) | `bg-brand-surface-3` + `shadow-lg` | Context menus, tooltips |

**Dark-mode shadow approach**: Use higher background luminosity + 1px border to create depth perception rather than drop shadows (which are invisible on dark backgrounds).

```css
/* Level 2 — modal */
.modal {
  background: var(--brand-surface-2);
  border: 1px solid var(--border-subtle);
  box-shadow: 0 20px 60px rgba(0, 0, 0, 0.5);
}
```

## 8. Component Catalogue

All components extend shadcn/ui primitives unless specified otherwise. Custom components wrap shadcn components — they never bypass them.

### 8.1 Button

```tsx
// Variants
<Button variant="default">    // Brand primary — CTAs
<Button variant="outline">    // Secondary actions
<Button variant="ghost">      // Tertiary, table row actions
<Button variant="destructive"> // Delete, cancel, void
<Button variant="link">       // Inline text actions

// Sizes
<Button size="sm">   // Table actions, compact forms
<Button size="md">   // Default
<Button size="lg">   // Primary page CTA

// With icon
<Button><Plus className="size-4 mr-2" />New Reservation</Button>

// Loading state
<Button disabled><Loader2 className="size-4 mr-2 animate-spin" />Saving</Button>
```

**Button rules:**
- Primary CTA per page: one only. Use `variant="default"`.
- Destructive actions must show a `ConfirmDialog` before executing.
- Icon-only buttons must have an accessible `title` or `aria-label`.

---

### 8.2 Input

All inputs are controlled via React Hook Form. Never uncontrolled.

```tsx
// Standard
<FormField
  control={form.control}
  name="guestName"
  render={({ field }) => (
    <FormItem>
      <FormLabel>Guest Name</FormLabel>
      <FormControl>
        <Input placeholder="Enter full name" {...field} />
      </FormControl>
      <FormMessage /> {/* Zod error message */}
    </FormItem>
  )}
/>
```

**Input rules:**
- Every input must have a visible `<FormLabel>`.
- Placeholder text supplements but never replaces the label.
- Error state: red border + red error message below (via `<FormMessage />`).
- Disabled state: 40% opacity, `cursor-not-allowed`.

---

### 8.3 Select / Combobox

- Use `<Select>` (shadcn) for static short lists (< 10 items).
- Use `<Combobox>` (shadcn Command) with search for long/dynamic lists (guests, rooms, employees).
- Combobox must support keyboard navigation.

---

### 8.4 DataTable

The primary component for listing data in the internal platform.

**Features:**
- Column sorting (click header)
- Server-side pagination (page, pageSize controls)
- Column visibility toggle
- Row selection (checkbox)
- Bulk actions toolbar (appears on row selection)
- Optional inline row actions menu (Edit, View, Delete)
- Loading skeleton (replaces data during fetch)
- Empty state (when no data matches)
- Responsive: horizontal scroll on tablet/mobile

```tsx
<DataTable
  columns={columns}
  data={reservations}
  isLoading={isLoading}
  pagination={pagination}
  onPaginationChange={setPagination}
  emptyState={<EmptyState icon={Calendar} title="No reservations found" />}
/>
```

### 8.7 Modal / Dialog

- Use `<Dialog>` (shadcn) for all modals.
- Max width: `max-w-lg` for forms, `max-w-2xl` for complex forms, `max-w-4xl` for data-heavy views.
- Always include a clear `<DialogTitle>` for accessibility.
- Keyboard: Escape to close, Tab trapping within modal.
- Do not stack more than one modal level deep.

### 8.8 Drawer

- Use `<Sheet>` (shadcn) for side drawers.
- Right-side drawer for: quick-view, filters, detail panels.
- Width: 400px (default), 640px (detail views).

### 8.9 Toast Notifications

```tsx
// Use shadcn Sonner integration
toast.success('Reservation confirmed successfully.');
toast.error('Room is not available for the selected dates.');
toast.warning('Stock level is below minimum threshold.');
toast.info('Check-in reminder: 3 guests arriving today.');
```

**Toast rules:**
- Duration: 4 seconds for success/info, 6 seconds for error/warning.
- Max 3 toasts visible simultaneously.
- Appear at bottom-right (internal), bottom-center (public website).
- Never use toasts as the only form of error feedback for forms — inline field errors are primary.

### 8.10 Tabs

Use `<Tabs>` (shadcn) for multi-section page layouts.

```tsx
<Tabs defaultValue="details">
  <TabsList>
    <TabsTrigger value="details">Details</TabsTrigger>
    <TabsTrigger value="history">History</TabsTrigger>
    <TabsTrigger value="folio">Folio</TabsTrigger>
  </TabsList>
  <TabsContent value="details">...</TabsContent>
</Tabs>
```

**Tab rules:**
- Tabs should not be nested.
- Each tab must change the visible content, not navigate to a new page.
- URL hash should reflect the active tab for shareability (e.g., `?tab=history`).

## 9. Dashboard Widget System

### Widget Shell

Every dashboard widget uses a consistent shell:

```tsx
<DashboardWidget
  title="Today's Occupancy"
  icon={BedDouble}
  isLoading={isLoading}
  error={error}
  permission="rooms:VIEW"
>
  {/* Widget content */}
</DashboardWidget>
```

The widget shell:
- Checks the `permission` prop — renders nothing if user lacks it.
- Shows skeleton loader while `isLoading = true`.
- Shows an inline error state if `error` is present.
- Has a consistent card frame.

## 10. Motion Principles

Motion communicates state, not decoration.

### Animation Tokens

| Token | Duration | Easing | Usage |
|---|---|---|---|
| `duration-instant` | 0ms | — | No animation (reduced-motion) |
| `duration-fast` | 100ms | `ease-out` | Button hover, badge appear |
| `duration-normal` | 200ms | `ease-out` | Modal open/close, dropdown |
| `duration-slow` | 350ms | `ease-in-out` | Page transition, drawer |
| `duration-skeleton` | 1500ms | `ease-in-out` | Skeleton shimmer loop |

### Framer Motion Conventions

```tsx
// Standard modal animation
const modalVariants = {
  hidden: { opacity: 0, scale: 0.96, y: 8 },
  visible: { opacity: 1, scale: 1, y: 0, transition: { duration: 0.2, ease: 'easeOut' } },
  exit: { opacity: 0, scale: 0.96, y: 8, transition: { duration: 0.15 } },
};

// Page fade-in
const pageVariants = {
  hidden: { opacity: 0, y: 12 },
  visible: { opacity: 1, y: 0, transition: { duration: 0.25, ease: 'easeOut' } },
};

// Staggered list items
const containerVariants = {
  visible: { transition: { staggerChildren: 0.05 } },
};
const itemVariants = {
  hidden: { opacity: 0, y: 8 },
  visible: { opacity: 1, y: 0 },
};
```

### Motion Rules

1. **Honor `prefers-reduced-motion`**: Wrap all Framer Motion animations with a `useReducedMotion()` check and skip animations when true.
2. **No motion on data tables**: Rows should not animate in — it slows scanning.
3. **Modals and drawers must animate**: Open/close transitions are expected.
4. **Toast animations**: Slide in from bottom-right, fade out.
5. **Skeleton shimmer**: Horizontal shimmer sweep, 1.5s loop, no bounce.
6. **Status changes on live data (Socket.IO)**: Brief highlight pulse (`ring-2 ring-brand-primary`) on updated rows.

## 12. Accessibility Standards

### Contrast Requirements

| Element | Minimum Ratio | Target |
|---|---|---|
| Body text on surfaces | 4.5:1 (WCAG AA) | 7:1 (AAA) |
| Large text (18px+ bold) | 3:1 (WCAG AA) | 4.5:1 |
| UI components (buttons, inputs) | 3:1 | 4.5:1 |
| Status badges | 3:1 minimum | — |

> [!WARNING]
> The brand gold `#C9A84C` on `--brand-deep` (#0D1321) achieves a contrast ratio of ~8.5:1 — AAA compliant. Do not place gold text on `--brand-surface-3` without verifying contrast.

### Focus Management

- All interactive elements must have a visible focus ring.
- Focus ring: `outline: 2px solid #C9A84C; outline-offset: 2px;`
- Modals and drawers must trap focus within them when open.
- When a modal closes, focus returns to the trigger element.
- Never remove focus outlines with `outline: none` without a custom replacement.

### ARIA Requirements

- `<Dialog>`: must have `aria-labelledby` pointing to the dialog title.
- `<DataTable>`: must use `<table>`, `<thead>`, `<tbody>`, `<th scope="col">`.
- Icon-only buttons: `aria-label="Delete reservation"`.
- Status badges: `aria-label` with full status text.
- Loading skeletons: `aria-busy="true"` on the parent container.
- Form errors: `aria-describedby` linking input to error message.

### Keyboard Navigation

| Interaction | Key |
|---|---|
| Open/close modal | Enter / Escape |
| Move between form fields | Tab / Shift+Tab |
| Select dropdown item | Arrow keys + Enter |
| Close dropdown/popover | Escape |
| Table row actions | Enter on focused row, then Tab to actions |
| Sidebar navigation | Tab to navigate items |
| Combobox search | Type to filter, Arrow keys to navigate, Enter to select |

## 14. Error Handling Patterns

### Inline Field Errors (Forms)

```tsx
// Displayed below the input, in red, with the FormMessage component
<FormMessage />
// Renders: "Check-out date must be after check-in date."
```

### Toast Errors (Mutations)

- Brief, actionable message.
- Do not expose raw API error objects to users.
- Example: "Failed to confirm reservation. Please try again." _(not a stack trace)_

### Full-Page Error State

Displayed when a critical data fetch fails (e.g., page-level query):

```tsx
<ErrorState
  title="Could not load reservations"
  description="There was a problem connecting to the server."
  action={<Button onClick={refetch}>Try Again</Button>}
/>
```

### Empty State

Displayed when a query succeeds but returns no results:

```tsx
<EmptyState
  icon={Calendar}
  title="No reservations yet"
  description="Confirmed reservations will appear here."
  action={<Button><Plus className="size-4 mr-2" />New Reservation</Button>}

```

### 404 Page (Internal)

- Message: "Page not found."
- Link back to Dashboard.
- No stack traces visible to users.

### 403 Page (Insufficient Permissions)

- Message: "You don't have permission to access this page."
- Show the user's current role.
- Link back to Dashboard.

## 15. Public Website Design Language

The public website shares the token architecture of the internal platform but is a **distinct design language**: light-first, warm, editorial, and unhurried. Where the internal platform optimizes for scanning density, the public site optimizes for atmosphere, trust, and desire. This section is the complete specification for the customer-facing experience.

### 15.1 Design Ethos & Inspiration

Studied references: **Explora Journeys** (editorial restraint, full-bleed sea imagery, capacity-detail rhythm), **Rosa Hotels** (video heroes, italic display accents, contemplative spacing), **Kloster Schaan** (calm, historic understatement), and **Crafto Hotel & Resort** (booking-widget prominence, room-card grids, testimonial cadence).

Distilled into eight public-site principles — these override nothing in §1; they extend it for the marketing surface.

| Principle | What it means in practice |
|---|---|
| **Atmosphere over information** | The hero sells a feeling first. Copy is sparse; imagery and space carry the weight. |
| **Editorial, not templated** | Alternating asymmetric image/text blocks. Sections breathe; nothing feels like a CMS grid dump. |
| **Cinematic imagery** | Full-bleed, high-quality photography and video. Images are the primary visual material, never decoration. |
| **Quiet luxury type** | Serif display (with occasional *italic accent words*), letter-spaced overlines, generous line-height body. |
| **Unhurried motion** | Slow, graceful reveals as content enters. Nothing bounces, nothing rushes. |
| **Warm neutral canvas** | Cream and sand, not clinical white. Deep espresso for dramatic sections. Gold as the single accent. |
| **One clear action per section** | Every section resolves to one intent: *Reserve*, *Explore*, *View menu*, *Order*. |
| **Placeholders are first-class** | Content is CMS-managed. Every media slot renders a dignified placeholder (§15.5) — never a broken image. |

### 15.5 Image & Media System (CMS Placeholders)

> [!IMPORTANT]
> The public website is **CMS-driven**. At build time and for any unpopulated slot, **there is no real image**. Every image position MUST render through the `<MediaFrame>` component, which shows a dignified, on-brand placeholder — never a broken `<img>`, never a raw gray box, never layout shift.

#### `<MediaFrame>` — the mandatory media primitive

```tsx
<MediaFrame
  slot="home.hero"           // stable CMS slot id (also the placeholder label)
  ratio="21/9"               // locked aspect ratio — reserves space, prevents CLS
  src={cms?.home?.hero}      // S3 key or CDN url; undefined → placeholder
  alt="Lobby at dusk"
  priority                   // hero images only
  overlay="scrim-bottom"     // optional scrim for over-image text
/>
```

Behaviour:
- **Aspect ratio is always locked** via `aspect-ratio` → zero layout shift whether the image loads or not.
- **`src` present** → `next/image` with `object-cover`, lazy (except `priority`), `.webp`.
- **`src` absent (placeholder state)** → renders the placeholder treatment below.
- **Loading** → skeleton shimmer (§13) tinted to the warm palette.

#### Placeholder treatment (unpopulated CMS slot)

A calm, branded panel — signals "image goes here" without looking broken:

- Background: `--pub-stone` with a very subtle diagonal hairline texture.
- Centered: a Lucide `ImageIcon` (`size-8`, `--pub-ink-muted`, `strokeWidth={1}`).
- Below icon: the slot label + ratio in `pub-body-sm` mono-ish caps — e.g. `HOME · HERO — 21:9`.
- A thin `--pub-line-strong` inset border, `--radius-2xl`.
- Never animated; never colored beyond the neutral scale.

#### Aspect-ratio tokens (per usage)

| Slot type | Ratio | Notes |
|---|---|---|
| Hero (full-bleed) | `21/9` desktop, `4/5` mobile | Cinematic; mobile goes portrait |
| Room / experience card | `4/3` | Grid consistency |
| Editorial split image | `3/4` (portrait) | Tall, magazine feel |
| Gallery tile | `1/1` and `3/4` mixed | Masonry variety |
| Dining dish thumbnail | `1/1` | Menu items |
| Offer card | `16/9` | |
| Logo / avatar | `1/1` | |

#### Over-image scrims

Text over imagery always uses a scrim so copy stays ≥ 4.5:1:

| Scrim | Gradient |
|---|---|
| `scrim-bottom` | `linear-gradient(to top, rgba(26,23,18,.72) 0%, rgba(26,23,18,0) 55%)` |
| `scrim-full` | flat `rgba(26,23,18,.45)` (hero with centered text) |
| `scrim-none` | no scrim (decorative image, no text) |

### 15.6 Motion & Scroll-Reveal Language

Extends §10. The public site is permitted **richer** motion than the internal platform — but it is *slow and singular*, never playful.

| Token | Duration | Easing | Usage |
|---|---|---|---|
| `pub-reveal` | 700ms | `cubic-bezier(0.22, 1, 0.36, 1)` | Section content entering viewport |
| `pub-reveal-slow` | 900ms | same | Hero content, large imagery |
| `pub-hover` | 300ms | `ease-out` | Card lift, image zoom on hover |
| `pub-kenburns` | 12000ms | `ease-in-out` | Very slow hero image scale (1.0 → 1.06) |

Rules:
1. **Scroll reveal**: elements enter with `opacity 0 → 1` and `translateY(24px) → 0`. Trigger once, when ~15% in view. Stagger children by 90ms.
2. **Hero**: image performs a slow `pub-kenburns` scale; headline reveals with `pub-reveal-slow`; overline → heading → subtext → CTA stagger in that order.
3. **Card hover**: image scales to 1.04 within its frame (`overflow-hidden`), card lifts 4px, gold underline draws in on the title. All `pub-hover`.
4. **Sticky nav transition**: transparent-over-hero → solid cream on scroll past 80px (see navbar, §15.7).
5. **Honor `prefers-reduced-motion`**: disable ken-burns, reveals become instant opacity, no parallax. Non-negotiable.
6. **No motion on**: prices, availability results, form fields. Trust surfaces stay still.
